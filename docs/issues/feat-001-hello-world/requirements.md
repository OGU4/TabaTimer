# feat-001 要求仕様書: シミュレーターで Hello World

## 1. プロジェクト概要

### 1.1 何を作るのか

Even Hub アプリとしての TabaTimer の最初の実行可能バージョンを作る。内容は、公式シミュレータ(evenhub-simulator)のグラス画面に「Hello World」という文字列を表示するだけの最小アプリと、その起動手順書である。

### 1.2 なぜ作るのか

- 開発環境一式(Vite + TypeScript + 公式SDK + シミュレータ)が連携して機能することを、最小のアプリで実証するため
- 以降の案件(タイマー機能の実装)で再利用する開発・検証の土台(プロジェクト構成、起動手順、自動検証手段)を確立するため

### 1.3 誰が使うのか

- 開発者(ユーザーおよび Claude Code)。エンドユーザー向けの機能はまだ含まない

### 1.4 どこで使うのか

- このPC(Ubuntu 24.04.3 LTS、Node.js v24.11.1、X11 ディスプレイ環境)上のシミュレータ
- 実機(Even G2)・スマホ(Even App)は本案件のスコープ外

## 2. 用語定義

機能設計書・コード内でも以下の用語を使う。

| 用語 | 定義 |
|------|------|
| Even App | Even Realities のスマホ公式アプリ。本番環境では Even Hub アプリの WebView をホストする |
| シミュレータ | `@evenrealities/evenhub-simulator` 0.7.3。開発時に Even App の代わりに WebView をホストし、グラス画面を描画するデスクトップアプリ |
| グラス画面 | シミュレータが描画する 576×288 ピクセルのキャンバス(実機では G2 の HUD に相当) |
| ブリッジ | SDK の `EvenAppBridge`。WebView 内の Web ページとホスト(Even App / シミュレータ)の間の通信路 |
| コンテナ | グラス画面 UI の構成単位。text / list / image の3種がある |
| テキストコンテナ | 文字列を表示するコンテナ(SDK の `TextContainerProperty`) |
| dev server | Vite の開発用 HTTP サーバ。アプリの Web ページを配信する |
| automation API | シミュレータの HTTP 制御 API(`--automation-port` で有効化)。スクリーンショット取得・コンソールログ取得・入力送信ができる |
| 表示文字列 | 本案件で表示する文字列。正確に `Hello World`(半角スペース1個、感嘆符なし、大文字小文字この通り) |

## 3. 機能要求一覧

### FR-001: グラス画面への Hello World 表示

- **機能名**: Hello World 表示
- **概要**: アプリがシミュレータに読み込まれると、ユーザー操作なしでグラス画面に表示文字列を表示する
- **入力**: シミュレータによるアプリ URL(dev server)の読み込み(ユーザーの追加操作なし)
- **出力**: グラス画面のテキストコンテナに表示文字列が描画される
- **受け入れ基準**:
  1. シミュレータでアプリを読み込むと、グラス画面に「Hello World」が表示される
  2. 文字列全体がグラス画面(576×288)内に収まり、見切れがない
  3. automation API `GET /api/screenshot/glasses` で取得した PNG 画像にも同じ文字列が写っている。判定は確認者の目視で行い、OCR 等の自動文字認識は用いない(確認者は、実装後検証では Claude Code(画像読解)、手動テストではユーザー)
  4. シミュレータがアプリの読み込みを開始してから 10 秒以内に表示される

### FR-002: 起動手順のドキュメント化

- **機能名**: 起動手順書
- **概要**: ユーザーが自分の手で開発環境を起動し、FR-001 の表示を再現できる手順書を提供する
- **入力**: 手順書(Markdown ファイル)を読む
- **出力**: 前提条件、起動コマンド列、表示確認方法、停止方法が記載された手順書
- **受け入れ基準**:
  1. 手順書に記載されたコマンドを記載順に実行するだけで、FR-001 の受け入れ基準1の状態に到達できる
  2. 手順書には以下がすべて含まれる: 前提条件(必要なツールとバージョン)、依存パッケージのインストール手順、dev server の起動コマンド、シミュレータの起動コマンド、表示の確認方法、両プロセスの停止方法

### FR-003: 表示失敗時のエラーログ出力

- **機能名**: エラーログ出力
- **概要**: 表示処理が失敗した場合に、失敗箇所と理由を WebView のコンソールに出力する
- **入力**: 表示処理の失敗(ブリッジ取得の失敗、またはコンテナ作成の失敗)
- **出力**: `console.error` によるエラーメッセージ
- **受け入れ基準**:
  1. ブリッジを 8000 ミリ秒以内に取得できない場合、タイムアウトしたことと、シミュレータ経由でページを開く必要がある旨を含むエラーメッセージが出力される
  2. コンテナ作成 API の戻り値が成功(0)以外の場合、戻り値の数値とその意味(Invalid / Oversize / OutOfMemory)を含むエラーメッセージが出力される
  3. 基準1・2 で出力されたエラーメッセージは automation API `GET /api/console` で取得できる

## 4. 非機能要求

### 4.1 パフォーマンス

- アプリの読み込み開始から表示完了まで 10 秒以内(FR-001 受け入れ基準4と同一)

### 4.2 対応環境

- OS: Ubuntu 24.04.3 LTS(X11 ディスプレイ必須。`DISPLAY` 環境変数が設定されていること)
- ランタイム: Node.js v24.11.1 / npm 11.6.2
- シミュレータ: `@evenrealities/evenhub-simulator` 0.7.3(Linux x64)
- 上記以外の環境(macOS、Windows、実機)での動作は本案件では保証しない

### 4.3 信頼性

- 永続化するデータはない(設定・状態の保存は行わない)。プロセス終了によるデータ消失リスクは存在しない
- dev server またはシミュレータの異常終了時は、手順書記載のコマンドで再起動すれば復旧できること

### 4.4 セキュリティ

- dev server は localhost(127.0.0.1)からのアクセスのみを想定する。LAN への公開設定は行わない

## 5. 制約条件

### 5.1 使用必須のライブラリ・ツール

- `@evenrealities/even_hub_sdk` 0.0.10(公式SDK)
- `@evenrealities/evenhub-simulator` 0.7.3(公式シミュレータ)
- Vite(ビルドツール。メジャーバージョンは機能設計書で確定する)
- TypeScript(言語。バージョンは機能設計書で確定する)

### 5.2 使用禁止・導入しないもの

- UI フレームワーク(React / Vue を含む全種)は本案件では導入しない(vanilla TypeScript で実装する)
- 非公式の Even 関連ライブラリ(even-toolkit を含む)は導入しない(公式SDKのみで実現する)
- `@evenrealities/evenhub-cli` は導入しない(実機サイドロード・ストア提出用のため本案件では不要)

### 5.3 ネットワーク制約

- インターネット接続は依存パッケージのインストール時(npm)のみ使用する
- アプリの実行時は localhost 内の通信のみ(dev server ↔ シミュレータ)

### 5.4 ファイル・配置制約

- アプリのソースコードはリポジトリ直下の `app/` ディレクトリに配置する
- `tmp/` は使い捨ての調査用であり、本案件の成果物を置かない
