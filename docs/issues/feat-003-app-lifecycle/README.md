# feat-003: アプリの起動・終了処理の仕様化

## 概要

TabaTimer アプリの**起動処理(startup)と終了処理(shutdown)の仕様を明文化**し、公式 SDK が推奨する起動シーケンスのうち未実装だった **`onLaunchSource`(起動元の判別)** を新規に取り込む。あわせて、これまで feat-002 で暗黙に実装していた起動/終了の挙動(ブリッジ取得 → `createStartUpPageContainer` → ダブルタップ終了 → ライフサイクル後始末)を要求仕様書・機能設計書として確定させる。

本案件で「アプリのライフサイクル骨格」を固め、以降のインターバルタイマー本体(設定・カウントダウン)の実装が乗る土台とする。

## 背景

- feat-001(Hello World)・feat-002(デジタル時計)で、起動と終了の挙動はコード上に存在するが、仕様書として明文化されていない。今後タイマー本体を載せる前に、起動・終了の骨格を要求/設計として確定しておく。
- SDK 0.0.10 の README/型定義を調査した結果、公式推奨の起動シーケンスに **`bridge.onLaunchSource(callback)`**(起動元が `appMenu` / `glassesMenu` のどちらかを1回だけ通知)があり、現状コードは未使用だった。これを取り込む。

## スコープ(feat-003 で決定)

- **対象**: 起動/終了挙動の仕様明文化 + `onLaunchSource` の新規実装。
- **背面化(FOREGROUND_EXIT)時の挙動**: 「何もしない(更新ループ継続)」とする。`setInterval` は止めない。実機での省電力/BLE 帯域の検証は実機到着後(将来フェーズ)に回す。

Closed(2026-06-14 完了。シミュレータで起動表示・秒進行・ダブルタップ終了を手動テスト合格)
