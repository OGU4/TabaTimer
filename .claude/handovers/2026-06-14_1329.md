- **bug-001 で設定変更A(`Cache-Control: no-store` / `server.watch.usePolling`)**: 却下。再現実験で dev server のファイル監視は正常・シミュレータ再起動で最新取得も確認でき、真因は古い dev server プロセスの残存と判明。設定では直らない。

- **クロック逆行説**: 却下。`date` で確認したらクロックは正常に進行(逆行は出力汚染による誤認)。

## ハマりどころ

- **Claude Code のツール出力汚染が全編で多発**(最重要):
  - 実在ファイルが "No such file" になる / `Read`・`sed` が壊れた内容を返す
  - `grep` が同じ行を数千行出力する(例: `bridge = await waitForEvenAppBridge();` が2000行)
  - `curl /api/console` が同じ3件+偽の `cleanup done (reason=beforeunload)` 行を返し続ける
  - `/api/screenshot/glasses` が突然 `content-length: 0`(feat-002 時は 2520B 取れた)
  - subagent が「main.ts/CLAUDE.md は変更なし」と誤報告(実際はコミット済み)
  - 一時的に「46c3d23 でコミット済み」という虚報(実在せず。正は `7045e91`)
- これらにより自動検証が全滅。X11 キャプチャ(`xwd`)に逃げたが、それも cmp 結果が信頼できず。
- 対処として git・ファイル実体での最小確認を多用したが、それでも結論を誤りかけた。

## 学び

- **観測異常が複数出たら、対象(コード/シミュレータ)でなく Claude Code のツール層バグをまず疑う**。壊れた観測 × 長時間自走 は最悪。早めに区切ってユーザーに返す。→ メモリ `tool-output-corruption-suspect-harness`。
- **コミットはユーザーの実画面テスト合格を確認してから**。feat-002 で自律検証後に先走ってコミットし、その「目視確認」もキャッシュで Hello World を見ていた。→ メモリ `commit-after-verified-test`。
- evenhub-simulator は **ページロード時に dev server が配信した版を実行**。最新保証は「Vite キャッシュ削除+dev server 完全再起動+古いプロセス全 kill」。シミュレータ再起動だけでは古い dev server が旧版を配信し続ける(=bug-001)。
- X11 直接キャプチャ手段: `DISPLAY=:1 xwd -id <wid>`、`xdotool search --name "EvenHub Simulator"` で窓 ID 取得可。automation API が死んでいる時の代替になりうる(今回は出力汚染で活かせず)。
