## 決定事項

- 安いモデルで雑用(git commit/push)するスキルは **subagent 方式**で作る。frontmatter に `model:` を書くインライン方式は禁止(1M 課金ゲート)。→ メモリ `model-switch-skill-via-subagent.md` に記録済み。
- スキル名は `/git-commit`(`/git-commit` = コミットのみ、`/git-commit push` = コミット+push)。
- commit と push は独立。push は直前にコミットしたかに関係なく実行(push-only ケース対応)。
- push 判定は意図ベース+安全側デフォルト(判定不能なら `しない`)。部分文字列マッチ禁止。
- リポジトリ固有規約(ブランチ・メッセージ言語・trailer)はハードコードせず、対象リポジトリの CLAUDE.md と git log から読み取る。
- 残った低2件(frontmatter `description` / 末尾ルールの旧表現「引数に push」)は実害なしのため**据え置き**(ユーザー承認「git-commit はこれでよい」)。

## 捨てた選択肢と理由

- **`model: sonnet` インライン方式(旧 `/commit` の設計)** → 長い会話でモデル切替時に全文脈を引き継ぎ 1M 課金ゲートに当たるため不可。subagent(新しい最小文脈)に委譲する方式へ。
- **`/commit` の名前を維持** → 一般的すぎるため `/git-commit` に改名。
- **低2件(旧"引数push"表現)の整合修正** → 本文の詳細ルールが優先され実害なし。スコープ(高・中)外。ユーザーが「これでよい」と判断し見送り。

## ハマりどころ

- Codex はこの PC で bwrap サンドボックス初期化に失敗し、リポジトリ内ファイルを自力で読めない。→ 初回レビューは `{ cat ファイル; } | codex exec "..."`(stdin 添付)。再レビューは `codex exec resume --last` が **stdin を読まない**ため、プロンプト引数に `$(cat ファイル)` を埋め込む。
- `git diff --stat` / `--name-status` はデフォルトで **unstaged のみ**。事前 `git add` 済みの変更を見るには `--cached` が要る(Codex 2巡目で指摘 → 手順2に `git diff --cached` を追加)。

## 学び

- スキルは subagent と違い現在会話のコンテキスト上で動く。モデルだけ差し替えても巨大文脈が引き継がれ 1M 帯に入る。雑用は subagent 委譲が正解。
- commit/push を分離すると push-only(クリーンツリーで未送信コミットだけ送る)が自然に扱える。
- `git diff` のデフォルトは working tree vs index(unstaged)。staged を見るのは `--cached`、HEAD からの全変更は `git diff HEAD`。
