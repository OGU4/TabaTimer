- 実装の技術方針(第一候補): 起動時に createStartUpPageContainer() で時刻表示用テキストコンテナを1つ作成し、setInterval で textContainerUpgrade() を毎秒呼び content を現在時刻に差し替える。automation API のスクショで自動検証する。

## 技術的発見(SDK 一次資料より)

- 画面更新の3手段の使い分け(README で明確化):
  - createStartUpPageContainer() = 起動時に1回だけ(2回目以降は無効)。
  - rebuildPageContainer() = 初回ページ以降の更新・再構築は全部これ(createStartUp を再呼びしない)。
  - textContainerUpgrade() = テキストコンテナの部分更新(content 差し替え)。デジタル時計向き。
- textContainerUpgrade() フィールド: containerID / containerName(max16) / contentOffset / contentLength / content(max2000)。戻り値 Promise<boolean>。
- background keep-alive: SDK 0.0.10 Changelog に「Enhanced WebView background keep-alive capability」。常時表示アプリ(タイマー)に追い風。ただし詳細仕様は未確認。
- ライフサイクルイベント: OsEventTypeList に FOREGROUND_ENTER_EVENT(4) / FOREGROUND_EXIT_EVENT(5) / ABNORMAL_EXIT_EVENT(6) / SYSTEM_EXIT_EVENT(7)。前面/背面/終了が取れる。
- 更新頻度の制約: 画像は「頻繁に送るな・並行送信禁止・キュー方式」と明記。テキスト(textContainerUpgrade)には頻度制約の明記なし。ただし毎秒更新が滑らかか/ちらつかないかはシミュレータで実証が必要。
- デモ非同梱: SDK・シミュレータとも、ローカルに動くデモ/サンプルプログラムは無い(simulator は package.json / README.md / bin/index.js のみ)。README 内のコード例が事実上のサンプル。公式 Web(GitHub 等)のデモ有無は未確認。
- TextContainerProperty の content は起動時 max 1000 文字(転送効率のため短く推奨)、textContainerUpgrade の content は max 2000 文字。座標は 0-576 / 0-288。
